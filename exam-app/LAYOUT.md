# 시험지 콘텐츠 레이아웃 규칙

목표: **임의의 지문·문제 데이터**가 들어오면, 수능형 2단 시험지 위에 페이지·단 배치가 자동으로 잡힌다.  
수동으로 left/right를 하드코딩하지 않는다.

구현: `src/layout/` (`packSheet.ts`, `constants.ts`) + `ExamSheet` 측정 레이어.

---

## 페이지 구조

```
ExamSheet
  └─ Header (첫 장 / 속지)
  └─ Content (2단 + 가운데 세로선)
  └─ Footer (페이지 박스 + 저작권)
```

- 콘텐츠 영역에 들어가는 단위는 **지문(Passage)** 과 **문제(Question)** 두 종류뿐이다.
- 데이터 순서: 지문 → 그 지문의 문제들 → 다음 지문 → … (`passages[].questionIds`)

---

## 공통 규칙

### 1. 상단 여백 (`COLUMN_TOP`)

- 지문·문제 **모두** 동일한 상단 여백을 가진다. (현재 **14px**)
- 적용 위치:
  - 단의 **맨 위**에서 시작할 때
  - 지문이 **끝난 뒤** 같은 단에 문제가 붙을 때 (문제 위 여백 = `COLUMN_TOP`)
  - 지문이 **다음 단으로 이어질** 때, 이어진 조각의 시작도 같은 상단 여백

### 2. 지문 흐름

- 지문은 단 높이까지 **끝까지 채운 뒤** 넘친다. (단락 단위로 미리 끊지 않음)
- 한 단락이 단 끝에 걸치면 **글자 단위**로 잘라 단을 채우고, 나머지는 다음 단(또는 다음 페이지 단)으로 보낸다.
- 이어지는 조각:
  - 이전 단: 박스 **아래 테두리·아래 패딩 없음** (`open-bottom`), 세로선은 **단 하단까지** 연장
  - 다음 단: 박스 **위 테두리·위 패딩 없음** (`open-top`), 상단 여백은 `COLUMN_TOP`

### 3. 지문 직후 문제

- 지문이 끝난 단에 **남은 높이**가 있으면, 문제를 **바로 이어서** 배치한다.
- 이때 문제 위 여백은 **`COLUMN_TOP`** 이다.

### 3-1. 문제 직후 지문

- 같은 단에 문제가 끝난 뒤 **새 지문**이 이어지면 **고정 여백** `QUESTION_TO_PASSAGE_GAP`(=`MIN_QUESTION_GAP`)을 둔다.
- 지문이 **단을 넘어 이어질 때** (`open-top` 조각):
  - 위 여백은 **다시 붙이지 않는다**.
  - 새 단 시작의 **`COLUMN_TOP`만** 적용한다 (상단 줄 맞춤).
- 공간이 부족해 문제 다음 지문이 같은 단에 못 들어가면 → 다음 단으로 보내고, 그 단에서는 일반 단 시작(`COLUMN_TOP`)으로 시작한다.

### 4. 문제 배치

| 규칙 | 값 / 동작 |
|------|-----------|
| 한 단 최대 개수 | **3** (`MAX_QUESTIONS_PER_COLUMN`) |
| 문제 사이 최소 간격 | **112px** (`MIN_QUESTION_GAP`) — 이보다 작아지면 개수를 줄이거나 다음 단으로 |
| 2개 이상일 때 간격 | 남은 세로를 **문제 사이만** 균등 분배 (2개면 상단·하단 맞춤) |
| 1개일 때 | 위에서부터 배치, 아래는 비움 |
| 한 단 배치 | 문제에 대해 **단마다 한 번만** 패킹. 못 넣은 나머지는 다음 단 |
| 쪼개기 | 문제는 **단/페이지로 쪼개지 않음**. 통째로 다음 단으로 이동 |

### 5. 단·페이지 진행

- 단 순서: 각 페이지 **왼 → 오른**, 그다음 페이지 왼 → …
- 첫 장 / 속지 헤더 높이가 달라 **콘텐츠 가용 높이**가 다르다 (`contentHeight('first' | 'continued')`).

## 자동 배치 파이프라인

```
1. Measure  지문·문제 DOM 높이 측정 (단 너비와 동일 스타일)
2. Pack     packSheet() — 위 규칙으로 페이지·단·조각 결정
3. Render   SheetColumn + PassageBlock / QuestionBlock
```

- 측정은 `ExamSheet`의 숨은 MeasureLayer + 지문 조각 프로브(`measurePassage`)로 한다.
- 패킹 결과는 `PackedPage[]` → 페이지 수·푸터 총 페이지에 반영된다.

## 상수 (`src/layout/constants.ts`)

| 상수 | 의미 | 현재 |
|------|------|------|
| `COLUMN_TOP` | 단 시작·지문 직후 문제 공통 상단 여백 | 14 |
| `MIN_QUESTION_GAP` | 문제 사이 최소 간격 | 112 |
| `QUESTION_TO_PASSAGE_GAP` | 문제 다음 새 지문 간격 (= `MIN_QUESTION_GAP`) | 112 |
| `MAX_QUESTIONS_PER_COLUMN` | 한 단 최대 문제 수 | 3 |
| `COLUMN_GUTTER` | 단 사이 거터(가운데 선 영역) | 22 |
| `COLUMN_INNER_PAD` | 단 안쪽 pr/pl | 4 |

여백·간격 조정은 이 파일만 바꾸면 된다.

---

## 데이터 계약 (요약)

```ts
passages: { id, label, intro, body, questionIds[] }[]
questions: { id, passageId, stem, choices, points, blocks?, … }[]
```

- `body`는 `\n\n`으로 단락 구분.
- 레이아웃 엔진은 **전체 목록을 순회**하며 지문→문제 순으로 단을 채운다.
- 새 지문/문제를 JSON에 추가하기만 하면 재측정·재패킹된다.

---

## 하지 않는 것

- 문제 본문을 단 중간에 잘라 이어 붙이기
- 단별 left/right 수동 지정
- 지문 박스와 가운데 세로선을 강제로 한 선으로 합치기 (거터 `COLUMN_GUTTER` 유지)
